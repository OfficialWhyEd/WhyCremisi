## 1. Perché la Memoria è Fondamentale

Un assistente senza memoria è un assistente che ricomincia ogni volta da zero. Risponde bene alle domande, ma non conosce il produttore. Non sa che preferisce kick asciutte. Non ricorda che il progetto "ALBUM_2025" è house music. Non sa che l'utente ha già rifiutato quel tipo di compressione tre volte.

La memoria trasforma uno strumento in un collaboratore.

`
  SENZA MEMORIA (tutti gli AI attuali)
  
  Sessione 1: "Come si usa il compressore?"  → risposta generica
  Sessione 2: "Come si usa il compressore?"  → risposta identica
  Sessione 100: identica alla sessione 1
  
  ──────────────────────────────────────────────────────────────
  
  CON MEMORIA OPENCLAW (WhyCremisi)
  
  Sessione 1: "Come si usa il compressore?"
              → risposta + salva: "utente è principiante, preferisce spiegazioni step-by-step"
  
  Sessione 2: "Come si usa il compressore sulla kick?"
              → risposta adattata: "Come ti spiegai prima, parti dal threshold..."
              → sa già che è house music, sa che usa FabFilter Pro-C2
  
  Sessione 50: capisce il contesto prima che l'utente parli
               → "Vedo che stai lavorando su una traccia simile a quella di
                  due settimane fa — vuoi usare la stessa catena di processing?"
`

### 3.2 Layer 1 — Memoria di Sessione

Persistita su disco come JSONL (JSON Lines). Ogni evento è una riga.

`
~/.whycremisi/sessions/
├── current.json           ← stato vivo della sessione corrente
├── 2025-05-12.jsonl       ← log eventi di oggi
├── 2025-05-10.jsonl
└── index.json             ← indice di tutte le sessioni
`

Formato evento JSONL:

`json
{"t":1716500010,"type":"ai.prompt","data":{"prompt":"analizza il mix","response_ms":640}}
{"t":1716500015,"type":"plugin.action","data":{"plugin":"FabFilter Pro-Q3","param":"Band1_Gain","from":0,"to":-4.5,"accepted":true}}
{"t":1716500020,"type":"user.feedback","data":{"rating":"positive","comment":"molto meglio"}}
{"t":1716500030,"type":"agent.observation","data":{"note":"utente accetta tagli EQ aggressivi, preferisce kick asciutte"}}
`

### 3.4 Layer 3 — Personalità Evolutiva

L'agente ha uno stile comunicativo che evolve in base ai feedback.

`cpp
// PersonalityCore.cpp — struttura
struct PersonalityState {
    std::string name = "WhyCremisi";
    std::string style = "professional_warm";
    float verbosity = 0.6f;       // 0=minimale, 1=dettagliato
    float technicality = 0.7f;    // 0=semplice, 1=tecnico
    float proactivity = 0.5f;     // 0=reattivo, 1=proattivo
    std::string language = "it";
    
    // Evolve con ogni sessione
    void evolveFromFeedback(const UserFeedback& fb);
};
`

Stili disponibili:

| Stile | Descrizione | Quando usato |
|-------|-------------|--------------|
| professional_warm | Preciso, diretto, con calore umano | Default |
| technical_minimal | Solo dati, nessuna narrativa | Utenti esperti |
| educational | Spiega ogni passaggio | Principianti |
| creative_partner | Suggerisce, propone, si entusiasma | Sessioni creative |
| quick_assistant | Risposte brevissime | Sessioni veloci |

## 4. Il Ciclo di Apprendimento

`
  ┌─────────────────────────────────────────────────────────┐
  │                  CICLO DI APPRENDIMENTO                 │
  └─────────────────────────────────────────────────────────┘
  
  PERCEZIONE          ELABORAZIONE         MEMORIZZAZIONE
  ─────────           ────────────         ──────────────
  
  Utente accetta   →  AgentSoul analizza  →  Aggiorna profilo
  suggerimento EQ      pattern: "accetta       acceptance_rate
                       tagli narrow <5dB"      eq_suggestions++
  
  Utente rifiuta   →  "Troppo aggressivo" →  Abbassa
  suggerimento         per questo utente       compression_depth
  comp
  
  Utente corregge  →  "Preferisce 128Hz   →  Aggiorna
  frequenza EQ         non 200Hz"              eq_frequency_bias
  manualmente
  
  Fine sessione    →  Distilla pattern    →  Aggiorna
                       principali             user_profile.json
                                              + session JSONL
`

## 5. Formato della Memoria in Prompt

Prima di ogni chiamata AI, WhyCremisi inietta il contesto in questo formato:

`
[CONTESTO SESSIONE]
Data: 2025-05-12 | BPM: 128 | Progetto: ALBUM_TRACK_07

[STATO AUDIO LIVE]
LUFS: -14.2 | Peak: -0.1dB | Larghezza stereo: 0.72
Problema rilevato: accumulo a 220Hz (-4dB ideale, +2dB attuale)

[PROFILO UTENTE]
Genere: House | Target LUFS: -14 | Kick: dry_tight
Plugin preferiti: FabFilter Pro-Q3, Pro-C2, Pro-L2
Accettazione EQ: 78% | Stile: professional_warm

[MEMORIA RILEVANTE]
- 3 sessioni fa: ha apprezzato taglio narrow a 200Hz sulla kick
- Sessione scorsa: ha rifiutato compressione ratio > 4:1 sul master
- Preferisce spiegazioni tecniche brevi

[PLUGIN ATTIVI]
Track 1 (Kick): FabFilter Pro-Q3 [slot 1] · Pro-C2 [slot 2]
Master: iZotope Ozone 11 [slot 1]

[PERSONALITÀ AGENTE]
Stile: professional_warm | Lingua: italiano
Verbosità: media | Tecnicità: alta
`

## 6. Privacy e Controllo dei Dati

`
  PRINCIPI DI PRIVACY
  
  ✓ Tutti i dati salvati localmente (~/.whycremisi/)
  ✓ Nessun dato inviato al cloud senza consenso esplicito
  ✓ /forget [argomento] — cancella memoria specifica
  ✓ /reset — azzera tutto il profilo utente
  ✓ Export JSON completo su richiesta
  ✓ La chiave API non è mai loggata in chiaro
`
